# Plan: Rediseño del módulo Documentos como Hub de Navegación

## Resumen

Transformar `DocumentosView` de un módulo con sub-vistas (lista + detalle de estudiante) a un **hub de navegación puro** que muestre solo registros existentes (causas + anotaciones) y redirija al flujo correspondiente al hacer click.

---

## Estado actual vs estado deseado

| Aspecto | Actual | Deseado |
|---------|--------|---------|
| Sub-view A (lista unificada) | Existe | Se mantiene |
| Sub-view B (detalle estudiante + generación) | Existe | **Se elimina** |
| Click en causa | Navega a CausasView | Sin cambio |
| Click en anotación | Muestra Sub-view B dentro de DocumentosView | Navega a AnotacionesView con estudiante pre-seleccionado |

### 3. `src/features/anotaciones/AnotacionesView.tsx` — Pre-seleccionar estudiante

Agregar un `useEffect` que lea `selectedStudentForAnnotations` del uiStore y pre-seleccione el estudiante:

```ts
import { useUIStore } from '../../shared/lib/stores/uiStore';

// Dentro del componente:
const selectedStudentForAnnotations = useUIStore((s) => s.selectedStudentForAnnotations);
const setSelectedStudentForAnnotations = useUIStore((s) => s.setSelectedStudentForAnnotations);

// Nuevo useEffect después de cargar students:
useEffect(() => {
  if (selectedStudentForAnnotations && students.length > 0) {
    const match = students.find((s) => s.id === selectedStudentForAnnotations);
    if (match) {
      setSelectedStudent(match);
      setSelectedStudentForAnnotations(null);
    }
  }
}, [selectedStudentForAnnotations, students]);
```

**Razón:** Reutiliza el flujo existente de AnotacionesView (tabla → modal de detalle) sin duplicar nada.

## Riesgos y mitigaciones

| Riesgo | Mitigación |
|--------|------------|
| Cross-navigation Anotaciones → Documentos (`selectedStudentForDocs`) deja de funcionar | Se mantiene intacto. El flujo actual de "Gestionar Documentos" en AnotacionesStudentDetailModal sigue funcionando — ahora mostrará el hub en lugar del detalle |
| Estudiante no encontrado en AnotacionesView al pre-seleccionar | El useEffect espera a que `students.length > 0` antes de buscar. Si no encuentra, simplemente no pre-selecciona |
| Performance: fetchStudentsWithAnnotationCounts se llama en ambos módulos | Aceptable — es una query ligera con RPC. Se podrían cachear en el futuro si es necesario |
