# Plan: Diagnóstico y Corrección de Subida/Procesamiento de Archivos PDF y MD

## Resumen Ejecutivo

El sistema de carga y procesamiento de archivos PDF/MD tiene **3 problemas de performance** que causan la demora percibida, **1 bug crítico** que corrompe datos, y **duplicación masiva** de código. Este plan corrige todo.

---

## Lo que la app extrae de los archivos

La app realiza este pipeline al subir un PDF/MD:

1. **Extracción de texto del archivo** (cliente):
   - MD → `FileReader.readAsText()` (texto plano directo)
   - PDF → `pdfjs-dist` extrae texto de cada página (ignora imágenes/escaneados)

2. **Subida a Supabase Storage** (bucket `anotaciones`)

3. **Llamada a IA** (OpenRouter, modelo `llama-3.1-8b-instruct`):
   - Envía el texto extraído con un prompt que le pide estructurar anotaciones
   - La IA devuelve un JSON array con campos: `text`, `date`, `type`, `registered_by`

4. **Registro en BD** (tabla `inspectorate_records`):
   - Cada anotación se inserta como fila con: `student_id`, `date_time`, `observation`, `severity`, `type`, `registered_by`, `tenant_id`, `pdf_file_path`

## Problemas Encontrados y Correcciones

### Fase 1: Fix crítico — Tipo "Información" convertido a "Negativa"

**Archivo:** `src/features/anotaciones/AnotacionesStudentDetailModal/hooks/usePdfProcessing.ts`
**Línea 212:**
```
ACTUAL:  type: a.type === 'Información' ? 'Negativa' : (a.type || 'Negativa')
CORRECTO: type: a.type || 'Información'
```
**Impacto:** Las anotaciones informativas se guardan incorrectamente como negativas, corrompiendo datos.

### Fase 2: Agregar timeout a llamadas HTTP a OpenRouter

**Archivos:**
- `server/lib/groq.ts` (dev) — usa `fetch`, agregar `AbortController` con 30s
- `server/api/lib/https.ts` (serverless) — usar `req.setTimeout(30000)` + `req.destroy()`

**Dev (`server/lib/groq.ts`):**
```typescript
const controller = new AbortController();
const timeout = setTimeout(() => controller.abort(), 30000);
try {
  const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
    ...,
    signal: controller.signal,
  });
  // ...
} finally {
  clearTimeout(timeout);
}
```

**Serverless (`server/api/lib/https.ts`):**
```typescript
req.setTimeout(30000, () => {
  req.destroy(new Error('OpenRouter timeout después de 30 segundos'));
});
```

### Fase 3: Aumentar max_tokens y mejorar regex JSON

**Archivos:** `server/lib/groq.ts` línea 16, `server/api/services/groq.ts` línea 22

- Cambiar `max_tokens: 2000` → `max_tokens: 4000` (documentos con >15 anotaciones se truncaban)
- Cambiar regex `\[[\s\S]*\]` (greedy) → `\[[\s\S]*?\]` (non-greedy) en los 3 archivos parse:
  - `server/routes/parse.ts` línea 76
  - `server/api/routes/parse.ts` línea 75
  - `api/index.js` línea 794
