# Workflow: Nuevo Feature Legal

## Descripción
Workflow completo para agregar una nueva funcionalidad legal al sistema.

## Pasos

### 1. Análisis
- [ ] Identificar requisito legal específico
- [ ] Buscar normativa aplicable
- [ ] Verificar si ya existe implementación parcial
- [ ] Definir alcance y plazos

### 2. Diseño
- [ ] Definir schema de datos necesario
- [ ] Diseñar flujo de usuario
- [ ] Definir reglas de negocio
- [ ] Planificar tests

### 3. Implementación Backend
- [ ] Crear/actualizar tabla en Supabase
- [ ] Agregar endpoints API
- [ ] Implementar lógica de negocio
- [ ] Agregar validaciones legales

### 4. Implementación Frontend
- [ ] Crear/actualizar componentes
- [ ] Integrar con API
- [ ] Agregar feedback al usuario
- [ ] Manejar estados de carga/error

### 5. Testing
- [ ] Tests unitarios
- [ ] Tests de integración
- [ ] Tests E2E
- [ ] Verificar plazos legales

### 6. Documentación
- [ ] Actualizar AGENTS.md
- [ ] Documentar endpoints
- [ ] Documentar reglas de negocio
- [ ] Agregar a checklist legal

### 7. Deploy
- [ ] Ejecutar lint
- [ ] Ejecutar tests
- [ ] Deploy a Vercel
- [ ] Verificar en producción
